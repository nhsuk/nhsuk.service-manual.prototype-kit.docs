How to use layouts

Layouts let you share a common design across pages. For example, to include the name of your service or use the NHS.UK footer on every page in your service.

If your pages share a custom header and footer, you can add them to one shared layout file. To change those parts of the page in future, you can change them once and they will update on all the pages that use that layout.

The Prototype Kit comes with a layout file for you to edit. You can also add more layouts if you need to.

Adding new layouts

In your code editor, copy the app/views/layout.html file, calling it something like app/views/layout-admin.html. You can then make any changes in this admin layout, for example having a different header or footer.

To use the new template in a view, change this line:

{{'{% extends 'layout.html' %}' | escape }}

To this:

{{'{% extends 'layout-admin.html' %}' | escape }}

You can make changes to existing blocks and define your own blocks in your layout.

Using blocks

Blocks are named sections of a page.

For example, in the NHS page template

there are blocks defined for header, content and footer.

You can use these in your layouts to set different areas of the page.

See the NHS page template options for a full list of all the defined blocks.

Stylesheets (CSS) and JavaScript

You can use custom layouts to load your own stylesheets (CSS) and JavaScript on multiple pages. Find out more about adding CSS and JavaScript.
