Some useful tips and links to help you with GitHub Codespaces.

Codespaces are a virtual computer in the cloud, when working in a Codespace you should imagine that the files in the Codespace are like the files on your computer.

Any changes you make are not shared with anybody else until you commit them to Git and push them to the repository.

The Codespace gives you a full code editor and user interface for managing your changes with github.

GitHub has a useful Quickstart guide to working with Codespaces

Things to remember when using Github Codespaces for NHS Prototypes

  1. GitHub Codespaces are automatically deleted after they have been stopped and have remained inactive for a defined number of days, by default this is 30 days.
  2. Any uncommitted and unpushed changes in a Codespace are lost when the Codespace is deleted. As they would be if changes were deleted on your computer before pushing them to the repository.
  3. It is not recommended to use Codespaces as the public host for your prototypes, for that please continue to use services like Heroku or Render. So that you can reliably password protect them.
