Learn when to use the NHS.UK prototype kit

The NHS.UK prototype kit lets you make HTML prototypes of NHS.UK services.

When to use the NHS.UK prototype kit

This kit lets you make HTML prototypes with the same front-end code as the NHS website. This means that you can:

  • create a service that looks and behaves like NHS online services, on mobile and desktop
  • test on the user‘s device and with assistive technology (for example, a screen reader)
  • show different pages based on user input (called 'branching') to test journey flows
  • share them by publishing on the web
  • make design decisions within the constraints of the web
  • share understanding of design ideas, flows and behaviours with the whole team – including those that use assistive technology
  • use common tools like Git and GitHub for collaboration and project history

If your service is in the alpha phase or beta phase and does not use the NHS.UK prototype kit for testing with users, you may get feedback in your service assessment report to use the kit.

Do not use the NHS.UK prototype kit for real services

You must not use prototype code for production. When your service is ready to move into production, the code needs to meet different standards. For example, it needs to be secure and perform well with large volumes of traffic.

When to use other prototyping tools

Sometimes you may want to use different tools for prototyping, for example if you:

  • want to draw different journeys as a team activity
  • need to make screens with people who are not comfortable using code
  • are designing screens that will not use the same components or patterns as the NHS website, for example clinician services

You can get examples of other community resources from the GOV.UK Design System. You may need to change these to use NHS.UK branding.
